# 動くお絵かきAR — Hiro Canvas録画版

Hiroマーカーを使って、魚・クラゲ・自分でアップロードした絵をARっぽく表示する静的Webアプリです。GitHub Pagesにそのまま置けます。

## 今回の修正

- 絵の表示をHTML画像要素ではなく、2D canvasの `drawImage()` に統一しました。
- 絵の幅は常に `高さ × 元画像の縦横比` で計算します。
- 高さスライダーは、絵のY位置だけを変えます。絵そのものの高さ比率は変えません。
- カメラを遠ざけても、Hiroマーカーの検出スケールを絵に反映しません。
- Hiro通知は小さく、短時間だけ表示されます。
- 録画ボタンを画面左上に常時表示しました。
- 録画ファイルには「実際のカメラ映像 + 絵」だけが入ります。
- menu、スライダー、トースト、録画ボタンは録画されません。

## ファイル

- `index.html` — アプリ本体
- `hiro-marker-print.html` — A4の約1/8サイズでHiroマーカーを印刷するページ

## GitHub Pagesで公開

1. このフォルダの中身をリポジトリ直下に置きます。
2. GitHub の Settings → Pages で、`main` / `/root` を公開元にします。
3. 公開URLをスマホで開きます。
4. `hiro-marker-print.html` を開いてHiroマーカーを100%で印刷します。
5. `index.html` を開き、カメラ許可を出してHiroマーカーを映します。

## 録画について

- 録画開始: 左上の「録画開始」
- 録画停止: 左上の「停止」
- 保存: 停止後に「動画を保存」

ブラウザによってはMediaRecorderの対応が違います。録画が使えない場合は、Chrome / Edge / Android Chromeで試してください。


## 録画ボタン表示の修正

- 録画UIは `index.html` 内に実装済みです。
- AR.js / A-Frame のカメラレイヤーに隠れないよう、録画UIを最前面の固定レイヤーにしました。
- iPhoneのノッチや安全領域に重ならないよう `env(safe-area-inset-*)` を使っています。
- 録画UIは画面には見えますが、録画canvasには描画しないため、保存動画には入りません。
