# 動くお絵かきAR — Hiroマーカー版

Hiroマーカー上で、魚・クラゲ・自分でアップロードした絵を動かすWeb ARアプリです。静的ファイルだけで動くので、GitHub Pagesにそのまま置けます。

## 今回の修正

- 操作メニューを右上の `menu` ボタンにまとめました。
- 設定はモーダルで表示されます。
- Hiroマーカーから離れたり、カメラが一時的にマーカーを見失ったりしても、画像がすぐ消えないようにしました。
  - 最後にHiroを認識した位置に画像を残します。
  - `表示位置をリセット` を押すと、次にHiroを映すまで非表示になります。
- 高さスライダーの上限は `5.0` のままです。
- Hiroマーカー印刷サイズは A4用紙面積の約1/8 に固定しています。
  - Hiro本体は `79mm × 79mm` です。
- 魚・クラゲ・自分の絵の縦横比をできるだけ保って表示します。

## 使い方

1. `index.html` と `hiro-marker-print.html` をGitHubリポジトリ直下に置く
2. GitHubの `Settings` → `Pages` で `Deploy from a branch` を選び、`main` / `/root` を公開元にする
3. 公開URLをスマホで開く
4. `hiro-marker-print.html` を開いてHiroマーカーを印刷する
5. `index.html` を開く
6. 右上の `menu` を押して、魚・クラゲ・自分の絵、動き、高さ、大きさを設定する
7. Hiroマーカーを映す

## Hiroから離れた時の動き

この版では、AR画像をHiroマーカーの子要素として直接置かず、Hiroの位置・向きだけを読み取っています。Hiroを認識したら、その位置を別の表示用アンカーにコピーします。

そのため、Hiroを見失っても画像は最後の位置に残ります。ただし、これは本格的な空間マッピングではないので、マーカーを見失った後にスマホを大きく動かすと、現実の場所と表示位置がずれて見えることがあります。もう一度Hiroを映すと位置が更新されます。

## 印刷のコツ

- 印刷倍率は `100%` または `実際のサイズ` にしてください。
- Hiro本体は `79mm × 79mm` です。
- A4の1/8サイズは便利ですが、認識距離は短くなります。
- うまく認識しない場合は、明るい場所で使う、紙を折らない、マーカー全体を画面に入れる、を試してください。
