# 動くお絵かきAR — Hiroマーカー版

Hiroマーカー上で、魚・クラゲ・自分でアップロードした絵を動かすWeb ARアプリです。静的ファイルだけで動くので、GitHub Pagesにそのまま置けます。

## 今回の修正

- 右上の `menu` ボタンはそのまま残し、絵の選択・動き・印刷・リセットをモーダルにまとめました。
- 高さ・横の位置・大きさ・回転は、画面下のスライダーでAR画面を見ながら調整できます。
- Hiroマーカーの検出ズレや距離変化で、絵の縮尺が変わらないようにしました。
  - Hiroから取得するのは位置と回転だけです。
  - 表示する絵は、縦横比を保ったまま一律スケールで拡大縮小します。
- Hiroの再検出・見失いトーストは削除しました。
- Hiroを見失っても、画像は最後に認識した位置に残ります。
- Hiroマーカー印刷サイズは A4用紙面積の約1/8 に固定しています。
  - Hiro本体は `79mm × 79mm` です。

## 使い方

1. `index.html` と `hiro-marker-print.html` をGitHubリポジトリ直下に置く
2. GitHubの `Settings` → `Pages` で `Deploy from a branch` を選び、`main` / `/root` を公開元にする
3. 公開URLをスマホで開く
4. `hiro-marker-print.html` を開いてHiroマーカーを印刷する
5. `index.html` を開く
6. Hiroマーカーを映して位置を決める
7. 画面下のスライダーで、高さ・横の位置・大きさ・回転を調整する
8. 右上の `menu` から、魚・クラゲ・自分の絵、動き、印刷、表示位置リセットを操作する

## 調整について

- `高さ`: マーカー面からどれくらい浮かせるかを変更します。最大 `8.0` です。
- `横`: マーカー基準で左右位置を変更します。
- `大きさ`: 絵の縦横比を保ったまま、全体サイズだけを変更します。
- `回転`: マーカー面に対する向きを変更します。

## Hiroから離れた時の動き

この版では、AR画像をHiroマーカーの子要素として直接置かず、Hiroの位置・向きだけを読み取っています。Hiroを認識したら、その位置を別の表示用アンカーにコピーします。

そのため、Hiroを見失っても画像は最後の位置に残ります。ただし、本格的な空間マッピングではないので、マーカーを見失った後にスマホを大きく動かすと、現実の場所と表示位置がずれて見えることがあります。もう一度Hiroを映すと位置が更新されます。

## 印刷のコツ

- 印刷倍率は `100%` または `実際のサイズ` にしてください。
- Hiro本体は `79mm × 79mm` です。
- A4の1/8サイズは便利ですが、認識距離は短くなります。
- うまく認識しない場合は、明るい場所で使う、紙を折らない、マーカー全体を画面に入れる、を試してください。
